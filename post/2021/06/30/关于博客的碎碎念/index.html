# 关于博客的碎碎念

📅 2021-07-16：初稿

# 为什么要写博客

在对博客进行折腾之前，我们确实需要思考一个问题，为什么要写博客，以便清晰地了解我们所需要的需求并进行取舍，可以有的放矢，避免陷入单纯为了优化而优化的怪圈。

关于学术博客的作用，有许多作者进行了讨论，我将其总结为以下几点：

  • 学习目的
    • 将输入转换为输出，费曼学习法的实践
    • 练习我们的沟通技巧
  • 交流目的
    • 与其他人分享想法，并参与一些前沿的讨论
    • 获得有价值的反馈
    • 练习我们的沟通技巧
  • 社交目的
    • 拓宽我们的职业社交网络
    • 提升我们在某领域的声誉

所以总结到底，博客的作用就是让我们尝试将自己的知识进行输出，以获得社交、交流、学习价值。同理，b站视频、短视频的制作也是同理（目前还没这个功夫制作）。

在这里的话，我目前还是侧重于学习价值，也就是费曼学习法的实践，尝试将所学的知识用自己的话再说一遍。

当然同时也需要兼备一定的交流作用，以增加自己认真书写的动力，同时也和平时记录markdown笔记的目的区分开来。【二者区别就是markdown笔记是初稿，写给自己的，自己能初步看懂就行，博客还是需要反复斟酌说下人话的。】

那么接下来就需要思考，自己写的内容需要博客具有什么样的功能。

在大多数情况下，写的应该都是计算机和数学应用相关的文章【包括但不限于机器学习、生物统计、生物信息、网页前端、balabala】，那么就需要两个最重要的功能：

  • 支持代码显示
  • 支持公式显示

很明显这个其实也就是markdown的语法功能嘛，所以就同样用markdown语言进行编写了，工具为所见即所得的typora。

# 博客平台和主题选择

这一部分博客的选择会因为自己水平的变化而有变化。

那么对于一个博客平台来说，有几个比较重要的特性：

  • 安全性，数据都能备份
  • 稳定性，浏览不会出问题
  • 美学性，好看就行
  • 易读性，排版舒服，有相关的标记语言
  • 简便性，修改便利

在这里的话，我一开始就打算自建博客，并没有选用CSDN、科学网、掘金这类的平台。

# halo 博客

最初采用的是halo博客，halo博客是采用java编写的，属于动态博客，具体可见halo官网。

动态博客的最大优点就是容易增删查改文章，例如halo博客内置了修改平台，可以直接进行修改。当然也有几个缺点（我的缺点）

  • 使用java编写，我不会
  • 需要用服务器挂载，之后不一定会续费服务器

所以最后我战略性地放弃了halo博客平台，将目光转向其他静态博客。

# vuepress

之后我将目光转向了vuepress，vuepress是由vue的作者尤雨溪开发的，最开始是作为项目文档的使用，和gitbook类似。

经过几年的发展，vuepress已经形成了一个良好的社区，社区内开发了许多主题，一些优秀的主题可以查看vuepress官方收录的[awesome-vuepress]。

使用vuepress的一个直观好处就是容易修改，由于它用的是vue编写的，由于具有一定的vue基础，所以很容易对主题和主题组件进行修改。另外一方面，组件的多元化也是我选择vuepress的重要原因。

此外，由于vue是JavaScript语言的继承，vuepress在交互式博客上具有得天独厚的优势，我一直想制作交互式博客，因此在vuepress进行博客搭建是十分符合我的要求的。

交互式博客

关于交互式博客我这里强烈大家浏览distill.pub的与互动文章交流，对网络上优秀的交互式博客进行了收集和点评，并提出了交互式博客的建议。我制作交互式博客的最初的启发是来自于Daniel Kunin的看见统计。

这里我采用的是vuepress主题gungnir，由邹笑寒大佬开发的，并在自己的喜好上进行了微调。

# 折腾记录

# vue图标引入

主题内置了 Vue 图标组件 oh-vue-icons作为图标支持。oh-vue-icons 集成了多个常用图标库，能让你轻松从不同的图标库引入 SVG 图标。

具体的图标可以进入Oh, Vue Icons! 或者是github项目地址查看

// 引入 OhVueIcon 组件
import OhVueIcon from "oh-vue-icons";

// 引入想要使用的图标
import { OiGitCompare, RiBook2Fill } from "oh-vue-icons/icons";
OhVueIcon.add(OiGitCompare, RiBook2Fill);

# CDN加速

又一次喜闻乐见的白嫖 jsDelivr，对于托管在 Github 上的博客来说，在静态资源路径前加上以下 URL 即可：

https://cdn.jsdelivr.net/gh/{{ 用户名 }}/{{ 仓库名 }}@{{ 分支名 }}

例如我的就是

https://cdn.jsdelivr.net/gh/chancey922/chancey922.github.io@main/
