# 什么是迁移学习

迁移学习是为了克服分立、隔离的学习模式，利用从某个任务得到的知识解决另一个相关的任务。和深度迁移学习方法相比，深度迁移学习直接提升了在不同任务上的学习效果，并且由于深度迁移学习直接对原始数据进行学习，所以它与非深度迁移学习方法相比有两个优势。

  • 能够自动化地提取更具表现力的特征

  • 满足了实际应用中的端到端（End-to-End）需求

传统的学习是孤立的，纯粹基于特定的任务、数据集和在它们上训练单独的孤立模型。不会保留可以从一种模型转移到另一种模型的知识。在迁移学习中，可以利用以前训练过的模型中的知识（特征、权重等）来训练新模型，甚至可以解决新任务数据较少等问题。

# fine-tuning

康奈尔大学的Jason Yosinski等人率先进行了深度神经网络可迁移性的研究[1]，该论文是一篇实验性质的文章Yosinski et al.，2014。对于ImageNet的1000个类别，作者将其分成两份（A和B），每份500个类别。然后，分别对A和B基于Caffe训练AlexNet网络。AlexNet网络共有8层，除第8层为与类别相关的网络无法进行迁移外，作者在1到7这7层上逐层进行微调实验，探索网络的可迁移性。 作者提出了两个有趣的概念：AnB和BnB。这里简单介绍一下AnB和BnB。（所有实验都是针对数据B来说的。）

  • AnB：将A网络的前n层拿来并将其固定，剩下的层随机初始化，然后对B进行分类。
  • BnB：将B网络的前n层拿来并将其固定，剩下的随机初始化，然后对B进行分类。

深度网络迁移的实验结果1如下。

先看BnB和BnB+（即BnB加上微调）。对于BnB而言，原训练好的B模型的前3层可以直接拿来使用，而不会对模型精度有什么损失。到了第4层和第5层，精度略有下降，不过还可以接受。然而到了第6层、第7层，精度居然奇迹般地回升了。 原因如下：对于一开始精度下降的第4层、第5层来说，到了这两层，特征变得越来越具体，所以下降了。而对于第6层、第7层来说，由于整个网络就8层，我们固定了第6层、第7层，那这个网络还能学什么呢？所以很自然地，精度和原来的B模型几乎一致。

通过实验得到了以下几个结论，这对深度学习和深度迁移学习的研究都有着非常好的指导意义。

（1）神经网络的前3层基本都是通用特征，进行迁移的效果比较好。

（2）在深度迁移网络中加入微调，效果提升会比较大，可能会比原网络效果还好。

（3）微调可以比较好地克服数据之间的差异性。

# ResNet简介

深度残差网络（Deep residual network, ResNet）的提出[2]是CNN图像史上的一件里程碑事件， 由Kaiming He、Xiangyu Zhang、Shaoqing Ren 和 Jian Sun在他们的优秀论文 Deep Residual Learning for Image Recognition 中首次提出 。让我们先看一下ResNet在ILSVRC和COCO 2015上的战绩：

ResNet的作者何凯明也因此摘得CVPR2016最佳论文奖

正如 VGG 作者已经指出的那样，表示的深度至关重要。在 VGG 的情况下，可以通过使用较小的卷积滤波器来构建更深的网络。

ResNet 的作者有一个聪明的想法：训练几个堆叠层的块来学习关于块输入的 残差函数，而不是在没有参考的上下文中学习通用函数网络架构。

在ResNet网络中有如下几个亮点：

  • 提出residual结构（残差结构），并搭建超深的网络结构(突破1000层)

  • 使用Batch Normalization加速训练(丢弃dropout)

在ResNet网络提出之前，传统的卷积神经网络都是通过将一系列卷积层与下采样层进行堆叠得到的。但是当堆叠到一定网络深度时，就会出现两个问题。

  1. 梯度消失或梯度爆炸。
  2. 退化问题(degradation problem)。

在ResNet论文中说通过数据的预处理以及在网络中使用BN（Batch Normalization）层能够解决梯度消失或者梯度爆炸问题。如果不了解BN层可参考这个链接。 但是对于退化问题（随着网络层数的加深，效果还会变差，如下图所示）并没有很好的解决办法。

20层与56层网络在CIFAR-10上的误差

所以ResNet论文提出了residual结构（残差结构）来减轻退化问题。下图是使用residual结构的卷积网络，可以看到随着网络的不断加深，效果并没有变差，反而变的更好了。

# fine-tuning步骤

  • 使用ResNet50预训练模型：该模型当初用ImageNet数据集训练时的输入图像尺寸是224x224。需要注意的是，使用预训练模型一定要确保让待训练的数据尽可能向原数据集靠拢，这样才能最大程度发挥模型的识图本领。
  • 预处理：按照预训练模型原本的预处理方式对数据进行预处理，采用tf.keras.applications.resnet50.preprocess_input
  • 基模型：导入预训练模型（只导入卷积层部分），并锁定全部卷积层参数
  • 定制模型：卷积层之后先接三个全连接层再接分类器，根据分类任务选择输出个数为2，用'softmax'激活
  • 优化器：采用较小学习率的SGD。
  • **Batch Size：**要用较小的batch size进行训练，这样可以让模型更快更好的收敛。这里采用了batch size=16
#预训练模型导入
conv_model = resnet50.ResNet50(weights='imagenet', include_top=False, input_shape=(224,224,3))
for layer in conv_model.layers:
    layer.trainable = False
    
#定制模型
x = keras.layers.Flatten()(conv_model.output)
x = keras.layers.Dense(100, activation='relu')(x)
x = keras.layers.Dense(100, activation='relu')(x)
x = keras.layers.Dense(100, activation='relu')(x)


predictions = keras.layers.Dense(2, activation='softmax')(x)
resNet_model = keras.models.Model(inputs=conv_model.input, outputs=predictions)
#优化器采用较小学习率的SGD
resNet_model.compile(optimizer='sgd', loss='categorical_crossentropy', metrics=['accuracy'])

num_epochs=20
# 训练模型
history = resNet_model.fit(train_generator,steps_per_epoch=STEP_SIZE_TRAIN,epochs = num_epochs,
                          validation_data = validation_generator,
                          validation_steps=STEP_SIZE_VALID,callbacks=callbacks)

# 图像预处理

使用要训练的各个模型的各个preprocess_input总是优于使用rescale = 1. / 255的方法，因为它将100％反映在训练预训练模型期间使用的预处理。

假设如果你是模型，然后你从小到大看的一千万张图片都是某一个亮度的，结果现在你被拽来看一堆新的图片，这些图片都巨亮无比，估计你的识图能力也会折损

API文档：tf.keras.applications.resnet50.preprocess_input
