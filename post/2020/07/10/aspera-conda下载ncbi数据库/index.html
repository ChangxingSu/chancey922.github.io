使用Aspera下载高通量测序数据

Chancey
2020-07-10
4 min

# 为什么选择Aspera

国内从NCBI下载高通量测序数据时候，由于数据量比较大，下载速度是个瓶颈，高速下载软件Aspera可以批量下载SRA文件，因此对其进行安装。

# 下载安装Aspera Connect

Aspera是IBM的快速下载工具，可以选择用压缩包安装和conda安装两种方式。我这里比较推荐用conda安装，因为在实际运行中发现aspera的版本有问题限制。

在这里，有一个小点就是，Aspera的安装是不能用root权限安装的，必须使用普通用户安装。

因此我们需要切换的普通用户进行conda安装。

首先先在conda中创造一个新的下载环境，然后激活环境

conda create -n download
conda activate download

在这个环境下安装相关包。

版本号的关联

在实际运行中发现能否成功，与perfetch和aspera的版本号十分密切。截止到2020年8月1日，以下版本的包是可以完美配合的。

conda install sra-tools=2.9.6 -y
conda install -c hcc aspera-cli=3.7.7

# 简单使用

安装完后，需要查看自己的安装路径

$ which ascp
/usr/local//anaconda3/bin/ascp

然后就可以在这个conda环境下进行下载了。

我们可以通过ascp的官方文件来查看相关参数信息

$ ascp -h
Usage: ascp [OPTION] SRC... DEST
          SRC to DEST, or multiple SRC to DEST dir
          SRC, DEST format: [[user@]host:]PATH
  -h,--help                       Display usage
  -A,--version                    Display version.
  -T                              Disable encryption
  -d                              Create target directory, implied for file/file-pair lists
  -p                              Preserve file timestamp
  -q                              Disable progress display
  -v                              Verbose mode
  -6                              Use IPv6
  -D...                           Debug level
  -l MAX-RATE                     Max transfer rate
  ···

主要用到的参数有这些

Aspera用法如下：
Usage: ascp [参数] 目标文件 保存路径
-v verbose mode 唠叨模式，能让你实时知道程序在干啥，方便查错。有些作者的程序缺乏人性化，运行之后，只见光标闪，压根不知道运行到哪了
-T 取消加密，否则有时候数据下载不了
-i 提供私钥文件的地址，不能少，地址一般是~/.aspera/connect/etc中的asperaweb_id_dsa.openssh文件
-l 设置最大传输速度，一般200m到500m，如果不设置，反而速度会比较低，可能有个较低的默认值
-k 断点续传，一般设置为值1
-Q 一般加上它
-P 提供SSH port，端口一般是33001，

因此我们根据参数进行设置。

ascp -k 1 -QT -l 300m -P33001 -i  /usr/local/anaconda3/envs/download/etc/asperaweb_id_dsa.openssh era-fasp@fasp.sra.ebi.ac.uk:/vol1/fastq/SRR103/004/SRR1039514/SRR1039514_2.fastq.gz .

可以看到下载速度就非常令人舒适了。
